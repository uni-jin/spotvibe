# 📝 SpotVibe 수동 테스트 체크리스트

이 문서는 실제 운영 환경에서 수동으로 테스트할 항목들을 정리한 것입니다.

**테스트 URL:** https://spotvibe-k.vercel.app  
**관리자 URL:** https://spotvibe-k.vercel.app/admin

---

## 🎯 필수 테스트 시나리오

### 시나리오 1: 초기 진입 및 지역 선택
1. ✅ 브라우저에서 `https://spotvibe-k.vercel.app` 접속
2. ✅ "Pick your Hotspot" 화면 표시 확인
3. ✅ 4개 지역 카드 표시 확인 (Seongsu, Hongdae, Hannam, Gangnam)
4. ✅ Seongsu만 "Available Now" 표시, 나머지는 "Coming Soon" 표시
5. ✅ Seongsu 카드 클릭
6. ✅ Feed 화면으로 전환 확인

**예상 결과:**
- 지역 선택 화면이 정상 표시됨
- Seongsu 클릭 시 Feed 화면으로 이동
- 하단 네비게이션 바 표시

---

### 시나리오 2: Feed 화면 기본 기능
1. ✅ Feed 화면 로드 확인
2. ✅ "Live Vibe Stream 🔥" 헤더 표시 확인
3. ✅ 선택한 지역명 표시 확인 (우측 상단)
4. ✅ Hot Spots Now 섹션 표시 확인
5. ✅ 카테고리 탭 표시 확인 (Pop-up Store, Restaurant, Shopping)
6. ✅ 기본적으로 "Pop-up Store" 탭 선택됨
7. ✅ 포스팅 카드들이 Pinterest 스타일로 표시됨
8. ✅ 각 포스팅 카드에 다음 정보 표시:
   - 썸네일 이미지
   - Vibe 상태 (✅ No Wait, ⏱️ Busy 등)
   - 장소명
   - 촬영 일시 (예: "Today 2:30 PM")
   - 좋아요 버튼 및 개수

**예상 결과:**
- Feed 화면이 정상 로드됨
- 포스팅들이 정상 표시됨
- 카테고리 탭이 정상 작동함

---

### 시나리오 3: 포스팅 상세 화면
1. ✅ Feed에서 포스팅 카드 클릭
2. ✅ 포스팅 상세 화면 표시 확인
3. ✅ 상단 헤더 확인:
   - 뒤로가기 버튼 (←)
   - 포스팅한 사용자 정보 (프로필 이미지, 이름)
   - 포스팅 시간 (예: "Today 14:30")
4. ✅ 이미지 carousel 확인:
   - 여러 이미지가 있으면 좌우 스와이프 가능
   - 이미지 영역에서 좌우 스와이프 시 상하 스크롤 비활성화
   - 이미지 영역 외부에서는 상하 스크롤 정상 작동
5. ✅ 이미지 하단 정보 확인:
   - Vibe 상태
   - 장소명
   - 촬영 일시
   - 설명 (있는 경우)
6. ✅ 좋아요 버튼 클릭 (로그인 시)
7. ✅ 뒤로가기 버튼 클릭 → Feed로 복귀

**예상 결과:**
- 포스팅 상세 화면이 정상 표시됨
- 이미지 carousel이 정상 작동함
- 스크롤 동작이 정상 작동함
- 좋아요 기능이 정상 작동함
- 뒤로가기가 정상 작동함

---

### 시나리오 4: 지도 화면
1. ✅ 하단 네비게이션에서 "Map" 클릭
2. ✅ 지도 화면 로드 확인
3. ✅ 상단 카테고리 필터 탭 확인 (Pop-up Store, Restaurant, Shopping)
4. ✅ 기본적으로 "Pop-up Store" 선택됨
5. ✅ 지도에 마커 표시 확인:
   - 관리자 등록 장소: 기본 Leaflet 마커
   - 사용자 포스팅: 썸네일 이미지 마커
6. ✅ 마커 클릭 시 팝업 표시 확인:
   - 썸네일 이미지
   - Vibe 상태
   - 장소명
   - 촬영 일시 (예: "Today 2:30 PM")
   - "View Detail" 버튼
7. ✅ "View Detail" 버튼 클릭 → 포스팅 상세 화면 이동
8. ✅ 다른 카테고리 탭 선택 → 해당 카테고리만 필터링됨

**예상 결과:**
- 지도 화면이 정상 로드됨
- 마커가 정상 표시됨
- 카테고리 필터가 정상 작동함
- 팝업 및 상세 화면 이동이 정상 작동함

---

### 시나리오 5: Post Vibe 기능 (로그인 필요)
1. ✅ Feed 화면에서 "+" 버튼 클릭
2. ✅ 로그인되지 않은 경우 → 로그인 모달 표시
3. ✅ "Sign in with Google" 클릭
4. ✅ Google 로그인 완료
5. ✅ 다시 "+" 버튼 클릭
6. ✅ Post Vibe 모달 열림 확인
7. ✅ 카테고리 선택:
   - "Pop-up Store" 선택 → 드롭다운에 팝업스토어 목록 표시
   - "Restaurant" 선택 → 드롭다운에 식당 목록 표시
   - "Shopping" 선택 → 드롭다운에 쇼핑 목록 표시
   - "Other" 선택 → 텍스트 입력 필드 표시
8. ✅ "Other" 선택 시:
   - 텍스트 입력 필드에 자동완성 제안 표시
   - 사용 횟수 표시 (예: "Cafe XYZ (3 uses)")
9. ✅ 이미지 업로드:
   - 메인 이미지 선택
   - 추가 이미지 선택 (선택사항)
10. ✅ Vibe 상태 선택 (Very Busy, Busy, No Wait, Quiet, Sold Out)
11. ✅ 설명 입력 (최대 500자, 선택사항)
12. ✅ "Post Vibe" 버튼 클릭
13. ✅ 성공 토스트 메시지 표시
14. ✅ Feed에 새 포스팅 표시 확인

**예상 결과:**
- 로그인 모달이 정상 작동함
- Google 로그인이 정상 작동함
- Post Vibe 모달이 정상 작동함
- 이미지 업로드가 정상 작동함
- 포스팅 저장이 정상 작동함

---

### 시나리오 6: 포스팅 삭제 (본인 포스팅만)
1. ✅ 본인이 작성한 포스팅의 상세 화면으로 이동
2. ✅ 우측 상단에 삭제 아이콘(🗑️) 표시 확인
3. ✅ 삭제 아이콘 클릭
4. ✅ 삭제 확인 팝업 표시 확인
5. ✅ "Delete" 버튼 클릭
6. ✅ 포스팅 삭제 확인
7. ✅ Feed에서 해당 포스팅 제거 확인

**예상 결과:**
- 삭제 아이콘이 본인 포스팅에만 표시됨
- 삭제 확인 팝업이 정상 작동함
- 포스팅 삭제가 정상 작동함

---

### 시나리오 7: 관리자 사이트 - 로그인
1. ✅ `https://spotvibe-k.vercel.app/admin` 접속
2. ✅ 로그인 화면 표시 확인
3. ✅ 사용자명: `super`
4. ✅ 비밀번호: `wlsdn123`
5. ✅ 로그인 버튼 클릭
6. ✅ 관리자 대시보드 표시 확인

**예상 결과:**
- 로그인 화면이 정상 표시됨
- 로그인이 정상 작동함
- 관리자 대시보드가 정상 표시됨

---

### 시나리오 8: 관리자 사이트 - 장소 등록
1. ✅ 관리자 대시보드에서 "장소관리" 메뉴 클릭
2. ✅ 장소 목록 표시 확인
3. ✅ "Add New" 버튼 클릭
4. ✅ 장소 등록 폼 표시 확인
5. ✅ 필수 필드 입력:
   - 카테고리: "Pop-up Store" 선택
   - 장소명 (영문): "Test Popup Store"
   - 위치 설정:
     - 지도 클릭하여 위치 선택
     - 또는 핀을 드래그하여 위치 조정
   - 활성화 여부: 체크
6. ✅ 선택 필드 입력:
   - 대표사진: 이미지 업로드
   - 설명: "Test description"
   - 노출기간: 시작일/종료일 설정 (선택사항)
7. ✅ "Save" 버튼 클릭
8. ✅ 저장 성공 확인
9. ✅ 장소 목록에 새 장소 표시 확인

**예상 결과:**
- 장소 등록 폼이 정상 작동함
- 지도에서 위치 선택이 정상 작동함
- 장소 저장이 정상 작동함

---

### 시나리오 9: 관리자 사이트 - 장소 검색
1. ✅ 장소관리 화면에서 검색 영역 확인
2. ✅ 카테고리 필터 선택 (예: "Pop-up Store")
3. ✅ 활성화 여부 필터 선택 (예: "Active")
4. ✅ 장소명 검색어 입력 (예: "Test")
5. ✅ 검색 결과 필터링 확인
6. ✅ 노출기간 검색:
   - 시작일 범위 설정
   - 종료일 범위 설정
7. ✅ 검색 결과 확인

**예상 결과:**
- 검색 기능이 정상 작동함
- 필터링이 정상 작동함

---

### 시나리오 10: GPS 기반 정렬 테스트
1. ✅ 브라우저에서 위치 권한 요청 확인
2. ✅ 위치 권한 허용
3. ✅ Feed 화면의 Hot Spots Now 섹션 확인
4. ✅ 각 장소 옆에 거리 정보 표시 확인 (예: "1.2km away")
5. ✅ 거리순으로 정렬되어 있는지 확인
6. ✅ 위치 권한 거부 시:
   - 거리 정보 미표시
   - 포스팅 수 → 최신 포스팅 시간 → 이름순으로 정렬

**예상 결과:**
- 위치 권한 허용 시 거리 정보 표시 및 거리순 정렬
- 위치 권한 거부 시 대체 정렬 로직 작동

## 🐛 오류 확인 체크리스트

### 콘솔 오류 확인
1. ✅ 브라우저 개발자 도구 열기 (F12)
2. ✅ Console 탭 확인
3. ✅ 빨간색 오류 메시지 확인
4. ✅ 경고(warning) 메시지 확인

**예상 결과:**
- 콘솔에 오류 없음
- 경고 메시지가 있어도 치명적이지 않음

### 네트워크 오류 확인
1. ✅ Network 탭 확인
2. ✅ 실패한 요청(빨간색) 확인
3. ✅ 404, 500 등의 오류 상태 코드 확인

**예상 결과:**
- 네트워크 요청이 정상적으로 완료됨
- 실패한 요청 없음

### RLS 정책 오류 확인
1. ✅ Supabase 관련 오류 메시지 확인
2. ✅ "row-level security policy" 오류 확인

**예상 결과:**
- RLS 정책 오류 없음
